# Acoustic Motor Fault Detector (TinyML)

Система акустической диагностики неисправностей электродвигателей в реальном времени на базе ESP32 и микрофона INMP441 с использованием алгоритмов машинного обучения (TensorFlow Lite for Microcontrollers).

## Описание проекта
Проект решает задачу классификации технического состояния электродвигателя по звуку его работы. Обученная нейронная сеть развернута непосредственно на микроконтроллере ESP32 (TinyML). Система захватывает звук через I2S-микрофон, потоково извлекает спектральные признаки (MFCC) и выдает вероятности по 6 диагностическим классам:

1. Healthy (Здоровый мотор)
2. Bearing Fault (Дефект подшипника)
3. Bow (Искривление ротора)
4. Broken Bars (Сломанные стержни ротора)
5. Misalignment (Несоосность)
6. Unbalance (Дисбаланс)

## 🛠 Аппаратное обеспечение
* **Микроконтроллер:** ESP32 
* **Микрофон:** INMP441 (всенаправленный цифровой микрофон, интерфейс I2S)

### Подключение INMP441 к ESP32:
| Вывод INMP441 | Вывод ESP32 | Описание |
| :--- | :--- | :--- |
| **SCK** | GPIO 25 | Serial Clock |
| **WS** | GPIO 26 | Word Select |
| **SD** | GPIO 33 | Serial Data |
| **L/R** | GND | Выбор левого канала |
| **VDD** | 3.3V | Питание |
| **GND** | GND | Земля |

## Machine Learning Pipeline (Python)
Пайплайн обучения модели реализован в Jupyter Notebook (`Acoustic_Motor_Fault_Detector_fixed.ipynb`).
* **Датасет:** HUST (акустические сигналы мотора).
* **Препроцессинг:** Понижение частоты до 16 кГц с использованием аппаратного anti-aliasing фильтра (`scipy.signal.resample_poly`). Устранена проблема утечки данных (Data Leakage) — аугментация шумом применяется исключительно к обучающей (Train) выборке *после* строгого разделения датасета.
* **Признаки (Features):** 80 признаков на основе Mel-frequency cepstral coefficients (Mean, Std, Max, Min от 20 коэффициентов).
* **Архитектура:** Сверхлегкая полносвязная (Dense) сеть с использованием BatchNormalization и Dropout (адаптирована для малых объемов данных).
* **Квантизация:** Модель конвертирована в TFLite с `int8` квантизацией, что уменьшило её размер до ~8 КБ и повысило обобщающую способность на независимом тесте до **83.3%**.

## Инференс на микроконтроллере (C++)
Код для ESP32 написан с использованием фреймворка Arduino под платформу PlatformIO.
Особенности реализации:
* **Потоковая обработка:** Звук пишется и обрабатывается скользящим окном по 400 сэмплов. Это избавляет от необходимости хранить весь 6-секундный аудиофайл в оперативной памяти.
* **DSP:** Программное удаление паразитного DC-смещения (DC Offset) микрофона и расчет MFCC-признаков на лету.
* **Экономия памяти:** Пиковое потребление RAM составляет около 56 КБ (Tensor Arena 40 КБ + FFT буферы 16 КБ).

## Как запустить проект

### Шаг 1: Обучение модели
1. Откройте ноутбук в Google Colab или локальном окружении Jupyter.
2. Запустите все ячейки (Run All).
3. Скачайте сгенерированный файл `motor_fault_model.tflite` и заголовочный файл `motor_fault_model.h`.
4. Из консольного вывода скопируйте сгенерированные C-массивы `MEAN` и `STD` и вставьте их в файл `normalization.h` в вашем C++ проекте.

### Шаг 2: Прошивка ESP32
1. Откройте проект в VS Code с установленным расширением **PlatformIO**.
2. Поместите файлы `motor_fault_model.h` и `normalization.h` в папку `src/`.
   > **Важно:** В файле `motor_fault_model.h` перед объявлением массива добавьте выравнивание памяти: `alignas(16) const unsigned char...`
3. Подключите ESP32 по USB, скомпилируйте и прошейте устройство.

### Шаг 3: Использование
1. Откройте Serial Monitor на скорости `115200`.
2. Дождитесь инициализации TFLite. Терминал перейдет в режим: `[ Ожидание команды... ]`.
3. Включите тестовый звук мотора.
4. Отправьте любой символ (например, нажмите `Enter`) в Serial Monitor для старта записи.
5. Микроконтроллер запишет 6.4 секунды аудио и выведет в терминал итоговый диагноз с вероятностями по каждому классу.
